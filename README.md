# P02b：CSMAR 上市公司财务特征清洗与分析

## 数据来源

- **数据库**：CSMAR（国泰安）
- **原始压缩包位置**：`data/data_raw_zip/`（7 个 `.zip` 文件，不上传 GitHub）
- **解压后原始数据位置**：`data/raw/`（不上传 GitHub）
- **样本范围**：A 股上市公司（沪深京），2000 年至最新可得年份
- **数据频率**：年度
- **实际有效窗口**：财务指标（资产负债表相关）最早可追溯至 **2011 年**，2000–2010 年财务比率缺失；股权结构、公司年龄等非财务指标自 2000 年起基本完整。

## 变量构造说明

所有比率型指标均采用**小数形式**（如 0.35 表示 35%）。  
由于数据缺少利润表字段，无法计算净利润，原要求中的 **ROA 与 ROE 分别以总资产（TA）和净资产（TE）替代**，并已在分析中注明。

| 变量 | 含义 | 计算方式 | 原始变量来源 |
|------|------|----------|-------------|
| `code` | 股票代码 | 6 位字符串，左侧补零 | 各表中股票代码列统一 |
| `year` | 会计年度 | 整数，从报表日期提取 | `accper`、`EndDate` |
| `lev` | 总负债率 | 总负债 / 总资产 | `FS_Combas-A002000000` / `FS_Combas-A001000000` |
| `sl` | 短期负债率 | 流动负债 / 总资产 | `FS_Combas-A002100000` / `FS_Combas-A001000000` |
| `ll` | 长期负债率 | 非流动负债 / 总资产 | `FS_Combas-A002200000` / `FS_Combas-A001000000` |
| `sdr` | 短期债务占比 | 流动负债 / 总负债 | `FS_Combas-A002100000` / `FS_Combas-A002000000` |
| `cash` | 现金比率 | 货币资金 / 总资产 | `FS_Combas-A001107000` / `FS_Combas-A001000000` |
| `ta` | 总资产（替代 ROA） | 直接取值 | `FS_Combas-A001000000` |
| `te` | 净资产（替代 ROE） | 直接取值 | `FS_Combas-A003000000` |
| `sloan` | 短期借款比率 | 短期借款 / 总资产 | `FS_Combas-A002101000` / `FS_Combas-A001000000` |
| `lloan` | 长期借款比率 | 长期借款 / 总资产 | `FS_Combas-A002201000` / `FS_Combas-A001000000` |
| `top1` | 第一大股东持股比例 | 原始值 / 100 | `Shrcr1` |
| `hhi5_approx` | 前五大股东持股集中度（近似） | (前五大持股合计 / 100)² | `Shrhfd5` |
| `size` | 公司规模 | ln(总资产) | `FS_Combas-A001000000` |
| `age` | 上市年龄 | 会计年度 − 上市年份 + 1 | `year`、上市日期 `LISTINGDATE` |

> **关于缺失指标的处理**：
> - `ROA` 与 `ROE`：缺少净利润，无法计算。分析中以 `TA` 和 `TE` 作为规模指标。
> - `HHI5`：作业要求的平方和需要各股东单独持股比例，本数据只有前五大合计，因此使用合计平方作为近似，并在分析中注明其局限。
> - 所有比率指标在清洗阶段已按年度 1% 和 99% 分位数缩尾，原始值以 `_raw` 后缀保留。

## 数据清洗说明

- **编码与读取**：Excel 文件使用 `openpyxl`；CSV 文件优先尝试 `gbk`，其次 `utf-8`。
- **主键统一**：股票代码统一转为 6 位字符串（`code`），年份统一为整数（`year`）。
- **日期处理**：所有日期列转换为 `datetime64` 类型。
- **样本筛选**：保留 `year >= 2000` 的观测。但资产负债表数据仅从 2011 年起有效，因此 2000–2010 年财务比率缺失。
- **报表口径**：数据默认均为合并报表口径（原始数据未提供 `Typrep` 字段区分类型）。
- **重复值处理**：按 `code`‑`year` 检查重复，保留第一条，并输出重复数量。
- **缺失值处理**：
  - 财务指标在 2011 年前系统性缺失（数据源限制），在图表和表格中保留为空白，并在分析中说明。
  - 2011 年后个别变量有少量缺失（如分析师关注度、机构持股），计算统计量时自动排除。
- **离群值处理**（winsorize）：
  - 对 `lev`, `sl`, `ll`, `sdr`, `cash`, `sloan`, `lloan`, `hhi5_approx` 进行**分年度 1%‑99% 缩尾**。
  - `ta`, `te`, `top1`, `size`, `age` 未缩尾。
  - 缩尾后变量名保持不变，原始变量以 `_raw` 后缀保留。
- **合并规则**：以“常用变量表”为主表，左连接“跨表查询表”和“公司信息表”，主键为 `code`‑`year`。每步合并后输出记录数，确保可追溯。
- **行业分类**：提取 CSMAR 行业代码首位字母作为行业大类（如 C39 → C）。**保留金融业（J）**，在分析中分组讨论。
- **日志记录**：关键处理步骤写入 `process_log.txt`。

## 存储方式

- **基础**：CSV 格式
  - `data/clean/firm_year_clean.csv`：清洗后的单表合并文件（未构造衍生指标）
  - `data/combined/csmar_firm_year_panel.csv`：最终公司‑年度面板（含所有构造指标及缩尾变量）
- **进阶**：Parquet 格式（列式存储）
  - `data/combined/csmar_firm_year_panel.parquet`（**不上传 GitHub**，可通过 Notebook 重建）
  - **选择理由**：Parquet 为列式存储，具有高压缩比和按列读取的优势。在本次百万行数据下，文件体积约为 CSV 的 1/4；当扩展至全部 A 股季度数据或多表关联时，性能优势更明显。内置 Schema 可避免 CSV 的类型推断错误。

## 如何运行

### 1. 安装依赖

在终端中执行以下命令：

```bash
pip install -r requirements.txt
```

主要依赖库包括：`pandas`, `numpy`, `openpyxl`, `scipy`, `pyarrow`, `matplotlib`, `seaborn`。

### 2. 将 CSMAR 原始压缩包放入 `data/data_raw_zip/`

从作业指定的云盘链接下载数据，把得到的 7 个 `.zip` 文件完整放入项目根目录下的 `data/data_raw_zip/` 文件夹。

### 3. 运行 `01_extract_raw_data.ipynb` 解压并识别原始数据

- 自动解压所有 `.zip` 文件到 `data/raw/`
- 扫描每个文件的列名，生成 `data/dict/variable_dictionary.csv` 变量字典

### 4. 运行 `02_clean_construct_variables.ipynb` 清洗数据并构造变量

- 统一股票代码和年度格式，处理缺失值、重复值，按年度 1%-99% 缩尾
- 构造全部财务指标和公司治理变量（`Lev`, `SL`, `LL`, `SDR`, `Cash`, `TA`, `TE`, `SLoan`, `LLoan`, `Top1`, `HHI5`, `Size`, `Age`）
- 合并多张表为公司‑年度面板，输出 `data/clean/firm_year_clean.csv` 和 `data/combined/csmar_firm_year_panel.csv`
- 生成缺失值统计表、缩尾对比表，并记录处理日志到 `process_log.txt`

### 5. 运行 `03_analysis.ipynb` 生成表格、图形和分析结果

- 年度描述统计表（`output/tables/yearly_summary.csv`）
- 趋势图：Lev 均值与中位数、总资产与现金比率双轴图
- 行业负债率分析：算术平均与加权平均趋势图、选定年份行业指标汇总表
- 股权结构箱线图（`Top1`）
- Parquet 进阶存储及性能对比演示

### 6. 打开 `report.html` 阅读完整报告

所有分析文本和图表已整合在报告中，可直接查看。
